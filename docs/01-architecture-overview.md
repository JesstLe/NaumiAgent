# 第一部分：架构总览

## 1. 设计哲学

基于 Anthropic "Building Effective Agents" 核心原则：

1. **保持简单** — 不引入不必要的抽象层，优先用最简模式解决问题
2. **透明可观测** — Agent 的每一步规划、决策、工具调用都可追踪
3. **精雕工具接口（ACI）** — 工具定义花和 prompt 工程同等的精力

### 核心取舍

| 维度 | 决策 | 理由 |
|------|------|------|
| 自建 vs 框架 | **自建核心 + 框架辅助** | Anthropic 建议直接用 LLM API，避免框架抽象层遮蔽调试 |
| 单 Agent vs 多 Agent | **单 Agent 为主 + 按需派生子 Agent** | 过早引入多 Agent 会增加 token 成本和调试复杂度 |
| 有状态 vs 无状态 | **有状态 + 持久化检查点** | 通用 Agent 需处理长任务，崩溃恢复是刚需 |
| 同步 vs 异步 | **异步优先** | 工具调用、子任务并行都需要异步执行 |

## 3. 核心模块职责

### 3.1 编排引擎（Orchestrator）

Agent 的"大脑"，基于 **Orchestrator-Workers** 模式（Anthropic 推荐）：

- **规划器（Planner）**：接收用户任务，分解为可执行的子步骤
- **执行器（Executor）**：按计划调用工具，收集结果
- **评估器（Evaluator）**：检查执行结果是否符合预期，决定是否需要重试或调整

编排模式根据任务复杂度自动选择：

```
简单任务 → 单次 LLM 调用（无工具）
中等任务 → Prompt Chaining / Routing
复杂任务 → Orchestrator-Workers + 并行子任务
需要质量保证 → Evaluator-Optimizer 循环
```

### 3.2 模型层（Model Layer）

通过 **LiteLLM** 统一接口，支持多模型路由：

| 模型 | 用途 | 选择理由 |
|------|------|---------|
| Claude Sonnet 4.6 | 主力推理、复杂规划 | 最佳编码/推理模型 |
| Claude Haiku 4.5 | 轻量任务、分类、路由 | 90% 能力，3x 性价比 |
| GPT-4o | 备选推理 | 多样化选择 |
| Gemini 2.5 Flash | 快速工具调用 | 速度优先场景 |
| Qwen3 / DeepSeek | 本地部署备选 | 成本敏感、离线场景 |

**路由策略**：
- 默认走 Claude Sonnet
- 简单分类/格式化走 Haiku
- 用户可配置模型偏好
- 支持按任务类型自动路由

### 3.3 工具层（Tool Layer）

基于 **MCP（Model Context Protocol）** 标准，Anthropic 主推的工具协议：

**内置工具集**：
- `file_read` / `file_write` / `file_edit` — 文件系统操作
- `bash_execute` — Shell 命令执行（沙箱内）
- `browser_navigate` / `browser_click` / `browser_screenshot` — 浏览器自动化
- `code_execute` — 代码沙箱执行（Python/JS）
- `web_search` / `web_fetch` — 网络信息获取

**扩展工具**：通过 MCP Server 接入 200+ 第三方服务（GitHub、Slack、数据库等）

### 3.4 记忆系统（Memory System）

三层记忆架构：

```
┌─────────────────────────────────────────────┐
│  工作记忆（Working Memory）                   │
│  - 当前任务的上下文窗口                        │
│  - 自动压缩（Context Compaction）管理          │
│  - 容量：128K tokens                         │
├─────────────────────────────────────────────┤
│  短期记忆（Short-term Memory）                │
│  - 当前会话的对话历史                          │
│  - 工具调用记录和结果                          │
│  - 存储：内存 + SQLite                        │
├─────────────────────────────────────────────┤
│  长期记忆（Long-term Memory）                 │
│  - 跨会话的用户偏好和知识                       │
│  - 历史任务摘要和经验                           │
│  - 存储：向量数据库（ChromaDB / pgvector）     │
└─────────────────────────────────────────────┘
```

### 3.5 安全护栏（Safety Guardrails）

四层防护：

1. **输入校验**：检测恶意指令注入、不安全请求
2. **输出审计**：过滤敏感信息泄露、有害内容
3. **预算控制**：单会话 token 预算上限、费用封顶
4. **权限管理**：工具调用白名单、文件访问路径限制、命令执行权限分级

## 5. 技术栈总览

```
语言:        Python 3.12+
异步框架:    asyncio + uvloop
模型接口:    LiteLLM（统一多模型）
工具协议:    MCP（Model Context Protocol）
编排引擎:    LangGraph（有状态图执行 + 检查点持久化）
记忆存储:    SQLite（会话） + ChromaDB/pgvector（长期）
沙箱执行:    Docker / E2B
浏览器自动化: Playwright
可观测性:    OpenTelemetry + LangSmith
CLI:        Typer + Rich
Web API:    FastAPI
配置:       Pydantic Settings（YAML + ENV）
包管理:     uv
测试:       pytest + pytest-asyncio
```
