## 商品详情页结构

### 1. 买家痛点

很多同学做 Agent 项目会卡在这几个地方：

- 项目只会调用一次大模型，面试官一问工具系统就说不清。
- 只做 RAG 或客服，项目同质化严重。
- 代码能跑，但不知道怎么写简历。
- 简历写得很高级，追问权限、记忆、评测、失败恢复时答不上来。
- 不知道 Agent 项目继续二开应该做什么。

这个资料包解决的是：让你把 Agent 项目从「能跑」升级到「能讲、能改、能验证」。

### 2. 项目亮点

这套项目围绕 NaumiAgent 的真实工程结构整理：

- Agent Runtime：ReAct 主循环、规划、工具执行、结果回写。
- Tool Registry：文件、Shell、浏览器、记忆、任务、调度等工具统一接入。
- Memory：SQLite 会话、长期记忆、上下文压缩。
- Safety：权限控制、预算、guardrails、危险操作边界。
- MCP：可以接第三方工具服务，不局限于本地函数。
- Browser Agent：浏览器观察、点击、截图、daemon 任务。
- Self-Evolve：自我审查、自我修改、工具锻造、目标追踪。
- Workbench 方向：本地优先、意图锁、决策日志、上下文健康度。

### 3. 你会收到什么

本商品为电子资料包，包含：

- 项目使用说明：怎么跑、怎么看、怎么演示。
- 7 天课程大纲：每天学什么、看哪些模块。
- 商品级架构讲解：Runtime、工具、记忆、安全、MCP、自进化。
- 三套简历写法：后端版、AI 应用版、Agent 工程版。
- 60 个面试问答：从基础到高阶追问。
- 30 条小红书内容矩阵：适合做个人账号记录或复盘。
- 项目讲解稿：30 秒、3 分钟、8 分钟版本。
- 二开路线：从新增工具到评测闭环、browser daemon、Workbench。
- 交付检查清单：避免拿了资料但不会落地。

### 4. 为什么它比普通 Agent 项目更适合面试

普通项目常见说法：

> 我接了大模型，然后让它回答问题。

这个项目可以讲：

> 我实现了一个本地 Agent Runtime，把模型调用、工具注册、权限判断、记忆召回、任务状态和运行态展示拆成独立模块。模型不是直接执行命令，而是通过 Tool Registry 发起结构化工具调用，经过 PermissionChecker 和 BudgetTracker 后再执行，并把结果回写到上下文和持久化会话里。

这就是差异。
