# 小红书笔记矩阵

使用方法：每天发 1-2 条，先发痛点和判断标准，再发项目结构，最后发转化内容。每条都不要硬广开头，先解决一个具体焦虑。

## 笔记 01：Agent 项目太 toy 了怎么改

- 开头钩子：很多 Agent 项目不是技术差，而是一眼就像 demo。
- 正文结构：先列 3 个 toy 信号：只调一次模型、没有工具边界、没有失败恢复；再给升级方向：Runtime、Tool、Memory、Safety。
- 结尾引导：想写进简历，先把「聊天」升级成「系统」。

## 笔记 02：适合写进简历的 Agent 项目应该长什么样

- 开头钩子：面试官不怕你项目小，怕你说不出工程判断。
- 正文结构：讲 5 层：入口层、Runtime、模型层、工具层、状态层；每层配一个可讲点。
- 结尾引导：项目越能拆层，越容易回答追问。

## 笔记 03：别再只写 RAG 客服了

- 开头钩子：RAG 不是不能写，是太多人只写到检索回答。
- 正文结构：对比 RAG demo 和 Coding Agent：工具调用、权限、记忆、执行状态、回放。
- 结尾引导：想差异化，就做能执行任务的 Agent。

## 笔记 04：后端同学转 Agent 开发，项目怎么选

- 开头钩子：后端同学做 Agent，不要丢掉自己的工程优势。
- 正文结构：后端优势是接口、状态、权限、任务、日志；把这些放进 Agent 项目。
- 结尾引导：Agent 项目本质也是复杂系统。

## 笔记 05：面试官问你的 Agent 怎么调用工具

- 开头钩子：不要只回答「模型会选择工具」。
- 正文结构：讲 Tool Schema、Tool Registry、Permission、执行结果结构化、上下文回写。
- 结尾引导：能讲链路，比只会调 API 强很多。

## 笔记 06：Agent 项目里最加分的不是 LLM

- 开头钩子：LLM 是底座，不是你的工程亮点。
- 正文结构：列出更加分的点：工具协议、权限控制、记忆压缩、任务状态、评测闭环。
- 结尾引导：简历上要写你设计了什么，而不是用了什么模型。

## 笔记 07：一个本地 Coding Agent 的架构拆解

- 开头钩子：如果你想做 Claude Code 类项目，可以先拆成这几层。
- 正文结构：CLI、Orchestrator、ModelRouter、Tool Registry、Memory、Safety、API。
- 结尾引导：先搭骨架，再堆能力。

## 笔记 08：为什么你的 Agent 项目面试讲不深

- 开头钩子：项目讲不深，通常不是功能少，而是没有边界。
- 正文结构：边界包括 UI/Runtime、Runtime/Tool、Tool/Safety、Session/Memory。
- 结尾引导：边界清楚，回答自然就清楚。

## 笔记 09：Agent 记忆系统怎么讲

- 开头钩子：不要把记忆说成「把聊天记录塞回 prompt」。
- 正文结构：短期会话、长期向量记忆、上下文压缩、召回注入、自动提取。
- 结尾引导：记忆不是存储，是上下文管理策略。

## 笔记 10：MCP 怎么写进简历

- 开头钩子：MCP 不要只写「接入了 MCP」，这样太空。
- 正文结构：讲 MCP Server 配置、工具发现、统一注册、权限边界、运行时接入。
- 结尾引导：写协议集成，不写名词堆砌。

## 笔记 11：Agent 权限系统怎么设计

- 开头钩子：让模型直接执行 Shell，是很多项目最危险的地方。
- 正文结构：读写分级、危险命令确认、路径限制、预算控制、审计日志。
- 结尾引导：安全边界是工程项目的高级感。

## 笔记 12：上下文压缩怎么讲才不虚

- 开头钩子：上下文不够用，不是简单截断历史。
- 正文结构：触发阈值、保留关键信息、写入长期记忆、恢复任务状态。
- 结尾引导：能讲压缩策略，就是比普通聊天项目深。

## 笔记 13：Agent 浏览器自动化有什么用

- 开头钩子：浏览器工具不是为了炫，是为了让 Agent 看见真实网页。
- 正文结构：goto、observe、click、screenshot、daemon、人工接管。
- 结尾引导：真实网页任务比纯文本任务更接近应用场景。

## 笔记 14：自我修改听起来玄，工程上怎么落地

- 开头钩子：自我进化不是让模型乱改自己。
- 正文结构：限定可改目录、保护核心模块、生成备份、运行验证、手动确认。
- 结尾引导：自进化的核心是边界和验证。

## 笔记 15：为什么要做 Tool Registry

- 开头钩子：工具越来越多时，if else 会先崩。
- 正文结构：统一 name、description、schema、execute；注册、发现、权限、渲染。
- 结尾引导：工具系统是 Agent 项目的骨架。

## 笔记 16：Agent 项目如何体现后端能力

- 开头钩子：后端能力不是只能写 CRUD。
- 正文结构：配置系统、SQLite、API、WebSocket、任务调度、权限模型、日志。
- 结尾引导：把后端能力翻译成 Agent 基础设施。

## 笔记 17：简历上 Agent 项目别这么写

- 开头钩子：「基于大模型实现智能问答」这句话太弱了。
- 正文结构：弱写法 vs 强写法；强写法强调模块、边界、指标、验证。
- 结尾引导：简历要写工程贡献，不写产品口号。

## 笔记 18：面试官问 ReAct，怎么答

- 开头钩子：ReAct 不是一句 Reason + Act 就结束。
- 正文结构：任务输入、思考、工具调用、观察结果、循环终止、失败处理。
- 结尾引导：把 ReAct 讲成执行状态机。

## 笔记 19：Agent 项目怎么做测试

- 开头钩子：AI 项目也要测试，不是跑一次 demo 就算。
- 正文结构：工具单测、权限单测、路由单测、CLI 命令测试、真实场景验证。
- 结尾引导：测试是区分项目和玩具的分水岭。

## 笔记 20：从 LangChain demo 升级到自研 Runtime

- 开头钩子：框架能帮你起步，但简历亮点要回到自己的设计。
- 正文结构：框架 demo 的问题；自研 Runtime 的价值；如何渐进迁移。
- 结尾引导：先理解框架，再抽象自己的核心链路。

## 笔记 21：一个 Agent 项目的 7 天学习路线

- 开头钩子：不要一上来读全项目，先按层拆。
- 正文结构：Day1 跑通；Day2 Runtime；Day3 Tool；Day4 Memory；Day5 MCP；Day6 自进化；Day7 面试。
- 结尾引导：学习项目要有顺序。

## 笔记 22：Agent 项目怎么回答失败恢复

- 开头钩子：面试官问失败恢复时，不要只说 try except。
- 正文结构：会话持久化、任务状态、后台任务、运行态面板、resume。
- 结尾引导：失败恢复是系统设计题。

## 笔记 23：为什么我推荐本地 Agent 项目

- 开头钩子：本地 Agent 更容易讲权限和工程边界。
- 正文结构：本地文件、Shell、浏览器、记忆、配置；比纯 SaaS demo 更容易演示。
- 结尾引导：本地系统更适合做工程深度展示。

## 笔记 24：Agent 项目二开应该先做什么

- 开头钩子：别急着加多 Agent，先补评测。
- 正文结构：新增工具、命令、评测脚本、运行日志、结果对比。
- 结尾引导：二开顺序决定项目质量。

## 笔记 25：如何把 Agent 项目讲给非 AI 面试官

- 开头钩子：不是所有面试官都熟 Agent，你要能翻译。
- 正文结构：把模型说成决策器，把工具说成能力接口，把记忆说成状态管理。
- 结尾引导：能翻译复杂概念，是表达能力。

## 笔记 26：Agent 项目和传统后端项目怎么结合

- 开头钩子：Agent 不是替代后端，而是多了一个决策层。
- 正文结构：API、任务队列、权限、日志、数据库、WebSocket 如何接 Agent。
- 结尾引导：用后端思维做 Agent，会更稳。

## 笔记 27：项目展示录屏怎么录

- 开头钩子：录屏别只展示聊天结果。
- 正文结构：展示启动、命令、工具调用、权限提示、运行态、测试。
- 结尾引导：录屏要证明你真的跑过。

## 笔记 28：Agent 项目答辩三件套

- 开头钩子：答辩时准备这三样就稳很多。
- 正文结构：架构图、运行截图、面试问答；每样怎么准备。
- 结尾引导：不要只带源码去面试。

## 笔记 29：如何判断一个 Agent 项目值得买

- 开头钩子：不要只看封面和功能数量。
- 正文结构：看源码结构、测试、运行说明、简历材料、二开路线、诚实边界。
- 结尾引导：能学会才是好资料。

## 笔记 30：我的工程级 Agent 项目资料包里有什么

- 开头钩子：这套不是代做，也不是复制粘贴包。
- 正文结构：列交付内容：课程、简历、面试、讲解稿、二开路线、检查清单。
- 结尾引导：适合愿意自己跑通并讲清楚项目的人。
