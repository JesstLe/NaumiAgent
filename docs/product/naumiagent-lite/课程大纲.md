# 7 天课程大纲

目标：让买家在 7 天内从「能跑项目」推进到「能讲架构、能写简历、能回答追问」。

## Day 1：跑通项目，建立全局地图

学习目标：

- 知道 NaumiAgent 是什么，不把它理解成普通聊天机器人。
- 能在本地启动 CLI 或 API。
- 能说出项目的主要目录职责。

建议操作：

```bash
uv sync --extra dev
uv run naumi
naumi serve
```

重点阅读：

- `README.md`
- `pyproject.toml`
- `src/naumi_agent/main.py`
- `docs/01-architecture-overview.md`

必须掌握：

- `orchestrator/` 是 Agent Runtime 主体。
- `tools/` 是能力层，不是 prompt 文案集合。
- `memory/` 负责会话、长期记忆、压缩。
- `safety/` 负责权限和预算。
- `api/` 提供 FastAPI 和 WebSocket 能力。

当天产出：

- 一张手写或电子架构图。
- 一段 30 秒项目介绍。

## Day 2：Agent Runtime 与 ReAct 主循环

学习目标：

- 能解释用户输入如何进入 Agent Runtime。
- 能讲清模型为什么不直接操作系统，而是通过工具。
- 能把 ReAct 讲成一个可控执行循环。

重点阅读：

- `src/naumi_agent/orchestrator/engine.py`
- `src/naumi_agent/orchestrator/planner.py`
- `src/naumi_agent/orchestrator/context_assembly.py`
- `src/naumi_agent/model/router.py`

讲解框架：

```text
用户输入 -> 上下文组装 -> 模型路由 -> 工具调用计划 -> 权限检查 -> 工具执行 -> 结果回写 -> 下一轮判断
```

当天产出：

- 3 分钟项目讲解稿第一版。
- 一段 ReAct 回答：为什么 Agent 需要观察工具结果后再继续决策。

## Day 3：Tool Registry 与权限系统

学习目标：

- 能解释工具的统一接口。
- 能讲 Tool Registry 解决了什么问题。
- 能解释权限系统为什么是 Agent 项目的核心亮点。

重点阅读：

- `src/naumi_agent/tools/base.py`
- `src/naumi_agent/tools/builtin.py`
- `src/naumi_agent/safety/permissions.py`
- `src/naumi_agent/safety/budget.py`
- `src/naumi_agent/safety/guardrails.py`

建议演示：

```text
/help
/runtime tools
/doctor
```

当天产出：

- 写出一个工具调用链路说明。
- 准备一个权限追问回答：为什么不能让模型直接执行 Shell。

## Day 4：Memory、Session 与上下文压缩

学习目标：

- 区分会话历史、长期记忆、上下文压缩。
- 能解释为什么 Agent 长任务需要持久化状态。
- 能讲清上下文压缩不是简单截断。

重点阅读：

- `src/naumi_agent/memory/session.py`
- `src/naumi_agent/memory/long_term.py`
- `src/naumi_agent/memory/compactor.py`
- `src/naumi_agent/memory/auto_extract.py`

建议演示：

```text
/history
/resume
/memory stats
```

当天产出：

- 一段「记忆系统」简历 bullet。
- 一个面试回答：长上下文不够时怎么处理。

## Day 5：MCP 与 Browser Agent

学习目标：

- 能解释 MCP 是什么，以及为什么比直接写本地函数更开放。
- 能讲浏览器工具为什么让 Agent 接近真实应用场景。
- 能区分内置 browser runtime 和外部 browser-debugging-daemon。

重点阅读：

- `src/naumi_agent/mcp/client.py`
- `src/naumi_agent/tools/browser/`
- `src/naumi_agent/tools/browser_daemon.py`
- `tests/unit/test_browser_daemon_tools.py`

建议演示：

```text
/browse https://example.com
/browser-state
/bdaemon health
```

当天产出：

- 一段 MCP 简历描述。
- 一段 Browser Agent 面试解释。

## Day 6：Self-Review、Self-Modify、Forge、Pursuit

学习目标：

- 能把自我演进讲成工程闭环，而不是玄学。
- 能解释自我修改为什么必须限制目录、备份、验证。
- 能讲目标追踪为什么需要成功标准和证据。

重点阅读：

- `src/naumi_agent/tools/self_modify.py`
- `src/naumi_agent/tools/self_evolve.py`
- `src/naumi_agent/tools/forge.py`
- `src/naumi_agent/orchestrator/pursuit.py`
- `src/naumi_agent/orchestrator/pursuit_store.py`

建议演示：

```text
/self-review src/naumi_agent/tools
/forge "生成一个分析日志关键词的工具"
/pursue "检查当前项目里一个可二开的功能点"
```

当天产出：

- 一段「自我演进」诚实表达：说明哪些已实现，哪些是继续二开的方向。

## Day 7：简历、面试与二开路线

学习目标：

- 选定自己的简历版本。
- 能完成 3 分钟项目讲解。
- 能回答 20 个基础问题和 10 个深挖问题。
- 选一个二开任务作为个人差异化证据。

建议操作：

```bash
uv run pytest tests/unit/test_engine.py -q
uv run pytest tests/unit/test_permissions.py -q
uv run pytest tests/unit/test_mcp_client.py -q
```

当天产出：

- 一版简历项目描述。
- 一张项目架构图。
- 一张运行截图。
- 一个二开 commit 或二开设计说明。

## 学完后的最低标准

你应该能做到：

- 30 秒讲清这个项目是什么。
- 3 分钟讲清核心架构。
- 说出 5 个真实源码目录。
- 回答工具调用、记忆、权限、MCP、失败恢复这些追问。
- 至少完成一个小二开或写出明确二开方案。
