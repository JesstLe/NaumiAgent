# 交付检查清单

这份清单用于卖家发货前自检，也用于买家学习后验收。

## 卖家发货前检查

- [ ] `README.md` 存在，并能说明资料包使用顺序。
- [ ] `商品页文案.md` 存在，并包含标题、卖点、价格、购买说明。
- [ ] `小红书笔记矩阵.md` 存在，并包含 30 条笔记。
- [ ] `课程大纲.md` 存在，并包含 7 天学习路线。
- [ ] `简历包装.md` 存在，并包含三套简历版本。
- [ ] `面试题库.md` 存在，并包含 60 个问答。
- [ ] `项目讲解稿.md` 存在，并包含 30 秒、3 分钟、8 分钟版本。
- [ ] `二开任务路线.md` 存在，并包含至少 6 个二开任务。
- [ ] 所有材料都能正常打开。
- [ ] 没有承诺录用结果、收入结果或生产级效果。

## 买家本地运行检查

建议买家至少完成这些动作：

```bash
uv sync --extra dev
python -m naumi_agent.main chat
```

进入项目后检查：

- [ ] 能看到 CLI 启动。
- [ ] 能执行 `/help`。
- [ ] 能执行 `/runtime`。
- [ ] 能执行 `/doctor`。
- [ ] 能说出 `src/naumi_agent/orchestrator` 的职责。
- [ ] 能说出 `src/naumi_agent/tools` 的职责。
- [ ] 能说出 `src/naumi_agent/memory` 的职责。
- [ ] 能说出 `src/naumi_agent/safety` 的职责。

## 学习成果检查

学习后应能完成：

- [ ] 写出一版 80 字项目描述。
- [ ] 写出 5 条简历 bullet。
- [ ] 画出 Runtime、Tool、Memory、Safety 四层架构图。
- [ ] 讲出 Tool Registry 的价值。
- [ ] 讲出权限系统为什么必要。
- [ ] 讲出长期记忆和上下文压缩的区别。
- [ ] 讲出 MCP 的作用。
- [ ] 讲出 browser daemon 的用途。
- [ ] 讲出自我修改的风险边界。
- [ ] 选择一个二开任务并说明验证方式。

## 面试准备检查

最低准备：

- [ ] 背熟 30 秒讲解稿。
- [ ] 能自然说完 3 分钟讲解稿。
- [ ] 从面试题库中挑 20 个基础题练习。
- [ ] 从面试题库中挑 10 个深挖题练习。
- [ ] 准备一张项目运行截图。
- [ ] 准备一张架构图。
- [ ] 准备一个测试命令输出。
- [ ] 准备一个自己改过或计划二开的功能点。

## 简历诚实检查

写简历前逐条确认：

- [ ] 写到的模块自己读过源码。
- [ ] 写到的命令自己本地跑过。
- [ ] 写到的测试自己执行过。
- [ ] 写到的二开任务自己做过或明确说明为规划。
- [ ] 不把雏形能力写成生产级能力。
- [ ] 不写「包过」「秒杀」「完全自主进化」这类表达。

## 商品交付边界

卖家应明确：

- 这是电子资料包，不是远程代做。
- API Key 和本地环境需要买家准备。
- 不承诺面试或录用结果。
- 可以提供文件缺失补发。
- 可以提供学习顺序建议。
- 如提供答疑，应单独说明答疑范围和时长。
