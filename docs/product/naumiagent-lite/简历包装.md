# 简历包装

使用原则：简历里的每一句都要能在源码、运行截图或本地验证中找到依据。不要把二开方向写成已经完成的生产能力。

## 项目名称建议

- NaumiAgent Lite：工程级本地 Coding Agent 系统
- 本地 Agent Runtime 与工具编排系统
- 面向代码任务的本地智能 Agent 平台

## 30 字版本

基于 Python 构建本地 Coding Agent，支持工具调用、记忆、权限、MCP、浏览器自动化与自我演进雏形。

## 80 字版本

设计并实现一个本地 Coding Agent 系统，包含 ReAct 编排引擎、统一工具注册、权限与预算控制、SQLite 会话、长期记忆、MCP 接入、浏览器自动化和自我审查/修改工作流，用于探索工程级 Agent 的运行时架构。

## 后端开发版

项目描述：

> 设计并实现本地 Agent Runtime，将模型调用、工具执行、权限控制、会话持久化和 API 服务拆分为独立模块。系统支持 CLI、FastAPI/WebSocket、多工具注册、SQLite 会话存储、后台任务和运行态面板，重点解决 Agent 长任务执行中的状态管理和安全边界问题。

可用 bullet：

- 基于 `Typer + Rich/prompt_toolkit + FastAPI` 构建多入口 Agent 应用，支持 CLI、单任务执行和 REST API。
- 设计 Tool Registry 抽象，统一文件、Shell、浏览器、记忆、任务、调度等工具的注册、Schema 和执行结果。
- 引入 SQLite 会话存储与运行态面板，支持历史会话、任务状态、后台任务和恢复链路。
- 实现权限与预算控制，将读操作、写操作、Shell、浏览器等能力纳入统一安全边界。
- 通过单元测试覆盖工具注册、权限、API、会话、记忆等关键模块，提升项目可维护性。

适合投递：

- 后端开发
- AI 应用后端
- 平台工程
- 工具链开发

## AI 应用开发版

项目描述：

> 构建一个面向代码任务的本地 Agent 系统，围绕 ReAct 循环实现模型路由、工具调用、上下文组装、记忆召回和结果回写。项目支持 MCP 扩展、浏览器自动化、自我审查和目标追踪，用于探索从 LLM 调用到 Agent 应用工程化的完整链路。

可用 bullet：

- 基于 LiteLLM 设计多模型路由，按 fast/capable/reasoning tier 组织模型能力。
- 实现 ReAct 风格执行链路，将用户任务拆分为模型推理、工具调用、观察结果和下一步决策。
- 设计上下文组装与长期记忆召回流程，支持会话恢复和上下文压缩。
- 接入 MCP 客户端和浏览器工具，使 Agent 能扩展到外部工具和真实网页任务。
- 实现 self-review、self-modify、forge、pursuit 等自我改进雏形，探索 Agent 反思与验证闭环。

适合投递：

## Agent 工程版

项目描述：

> 实现一个类 Claude Code/Codex 的本地 Agent 工程原型，核心包括 Agent Runtime、Tool Protocol、Memory、Safety、MCP、Browser Agent、Task/Pursuit 和 Workbench 治理方向。项目重点不在单次问答，而在 Agent 如何安全、可观测、可恢复地执行多步任务。

可用 bullet：

- 抽象 Agent Runtime 与 Tool Protocol，将模型推理和外部执行能力解耦，支持工具生命周期管理。
- 建立权限、预算、guardrails 三层安全机制，降低 Agent 执行文件和 Shell 操作的风险。
- 实现会话持久化、长期记忆、上下文压缩和运行态通知，使长任务具备恢复和审计能力。
- 接入浏览器自动化和 browser daemon 工具，支持网页观察、交互、截图与任务队列。
- 设计 Workbench 治理方向，包括本地优先、意图锁、决策日志、上下文健康度和验证报告。

适合投递：

- Agent Infra
- AI Developer Tools
- LLM Platform
- Coding Agent 方向

## 面试时的项目总结

可以这样说：

> 我做的不是一个单纯聊天项目，而是一个本地 Coding Agent Runtime。它把用户任务交给模型推理，但模型不能直接操作系统，而是通过统一 Tool Registry 调用文件、Shell、浏览器、记忆等工具。每次工具调用会经过权限和预算控制，结果会回写到上下文和会话存储。项目还做了 MCP、浏览器自动化、自我审查和目标追踪这些扩展，核心目标是探索 Agent 如何从 demo 变成可观测、可恢复、可二开的工程系统。
