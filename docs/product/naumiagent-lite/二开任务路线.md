# 二开任务路线

目标：让买家从「看懂项目」进阶到「拥有自己的改动证据」。每个任务都指向真实目录和验证命令。

## 任务 01：新增一个确定性文本分析工具

难度：简单

目标：

新增一个工具，用于统计输入文本的行数、字符数、关键词命中数。这个任务适合理解 Tool 基类和注册流程。

建议查看：

- `src/naumi_agent/tools/base.py`
- `src/naumi_agent/tools/forge.py`
- `src/naumi_agent/orchestrator/engine.py`
- `tests/unit/test_tool_registry.py`

预期输出：

- 新增一个工具类，包含 `name`、`description`、`parameters_schema`、`execute`。
- 新增或更新对应测试。
- 能在工具注册列表里看到它。

验证命令：

```bash
uv run pytest tests/unit/test_tool_registry.py -q
uv run ruff check src/naumi_agent/tools tests/unit/test_tool_registry.py
```

面试讲法：

> 我通过新增确定性工具理解了 Tool Protocol，工具不只是 prompt，而是有参数 Schema、执行逻辑和测试的能力单元。

## 任务 02：新增一个 slash command 调用已有工具

难度：中等

目标：

新增一个 CLI 斜杠命令，让用户可以手动触发某个已有工具。重点是复用底层 execute，不重复实现逻辑。

建议查看：

- `src/naumi_agent/main.py`
- `src/naumi_agent/cli/slash_router.py`
- `src/naumi_agent/cli/completer.py`
- `tests/unit/test_main_analysis_dispatch.py`

预期输出：

- 新命令出现在 help 或补全里。
- 命令逻辑调用已有工具。
- 对错误参数给出中文提示。

验证命令：

```bash
uv run pytest tests/unit/test_cli_completer.py -q
uv run pytest tests/unit/test_main_analysis_dispatch.py -q
```

面试讲法：

> 我没有为 slash command 复制一套逻辑，而是让用户手动命令和 Agent 工具调用复用同一套底层能力。
