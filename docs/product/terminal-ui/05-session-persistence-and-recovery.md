# 05 会话持久化与恢复

## 1. 目标

确保切换页面、切换会话、退出重开、Bridge 重启和意外崩溃后，用户的对话、草稿、运行状态和阅读位置都能按明确语义恢复，不出现“页面一切换就失败”或“消息消失”。

## 2. 双层持久化

### 后端权威层

SQLite 保存会话、消息、运行、工具结果、权限决议和完成收据。所有可影响执行判断的事实必须进入该层。

### 前端表现层

`.naumi/terminal-ui-state.json` 保存每个项目/会话的草稿、路由、Inspector 标签、折叠项、滚动锚点和输入历史游标。该文件损坏不得影响后端会话。

前端存储键使用 `repository_identity + session_id`，不再只依赖当前目录字符串。

## 3. 快照版本与写入

- 快照包含 `schema_version`、`updated_at`、`project_id`、`session_id`。
- 每个版本提供显式迁移函数；未知新版本只忽略表现状态，不删除文件。
- 写入采用临时文件、fsync 和原子 rename。
- 草稿编辑防抖写入；提交、切页、切会话和退出前立即刷新。
- 不持久化 token、权限凭据、完整工具敏感参数。

## 4. 重连与重放协议

客户端在 hello 时发送：协议版本、项目身份、会话 ID、最后确认 `seq` 和未确认请求 ID。服务端响应以下一种：

- `resume_delta`：从下一序号补发增量。
- `resume_snapshot`：发送规范化快照后继续增量。
- `resume_completed`：运行已在离线期间完成。
- `resume_unavailable`：说明原因并保留历史只读视图。

客户端按 `event_id` 去重，按 `seq` 排序；同一提交的 `request_id` 确保幂等，避免重连时重复执行任务。

## 5. 会话切换

切换前保存当前表现快照，切换后先渲染本地骨架，再用后端重放校正。旧会话仍在运行时继续接收轻量状态，完成或等待权限时在全局状态区提示。

用户切回后应看到连续时间线，不应出现重复用户消息、重复工具卡或临时空白 Agent 消息。

## 6. 崩溃恢复

1. 检测上次是否非正常退出。
2. 验证快照结构，隔离损坏条目。
3. 向 Bridge 查询最后会话和运行事实。
4. 合并后端事件与本地草稿，不自动重新提交草稿。
5. 若存在中断运行，展示恢复、放弃或只读查看选项。

恢复过程必须记录 debug trace，但用户界面只展示可行动摘要。

## 7. 边界场景

- 同一项目两个终端同时打开：各自拥有客户端 ID，后端事件共享，草稿不互相覆盖。
- Git 仓库移动或 worktree 切换：用仓库 identity 和规范路径识别，明确提示迁移。
- 会话被外部删除：保留只读本地草稿，返回会话列表。
- 磁盘只读/空间不足：继续运行但明确提示表现状态无法保存。
- 系统时钟回拨：排序以 `seq` 为准，不以时间戳为准。

## 8. 测试与验收

使用真实 SQLite 和真实 Bridge 覆盖切页、切会话、正常重启、kill -9、重复事件、序号缺口、双客户端、损坏 JSON、只读目录和离线完成。验收要求正在流式输出时切换页面或会话，返回后内容连续且光标位于草稿末尾。
