# 实施设计与计划记录

本目录保存 NaumiAgent 按日期形成的功能设计与实施计划：

- `specs/`：设计选择、边界、状态机和验收定义；
- `plans/`：当次实现的文件、测试和提交步骤。

这些文件是不可变的工程记录，不是当前用户手册。它们可能包含当时仍有效、后来已经退役的命令，
也可能描述目标态而非已交付能力。查看当前安装、入口、配置和 fallback 行为时，请回到
[NaumiAgent 文档地图](../README.md)。

## 阅读规则

1. 先看文件日期和标题，确认它对应的功能切片。
2. 设计中的“目标”“验收标准”不等于已经实现；需要结合后续提交和当前文档判断。
3. 不为更新现状而重写旧记录。新的决策通过新设计文件追加，并在当前文档中同步结果。
4. 旧 Prompt Toolkit CLI 的历史记录继续保留；其实现、测试和必要依赖也继续保留，但没有公共入口。
5. 发现历史记录与当前行为不同，不视为链接或文档错误；把旧命令复制到当前手册才是治理错误。

文档状态和检查范围由 [`docs/governance.json`](../governance.json) 管理。
