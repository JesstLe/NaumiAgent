# Shell Output Artifacts Design

## Goal

让前台 `bash_run` 在命令产生大量输出时不再静默丢失证据：模型和用户收到有边界、可读的首尾摘要，同时完整输出保存为受管日志，可继续用文件读取工具检查。

## Existing Behavior

- NaumiAgent 已有真实 Shell 工具 `bash_run`，通过跨平台 shell runtime 创建独立进程组，并在超时或取消时回收进程树。
- 长任务已有独立的 `background_run` 生命周期，不需要再造第二套后台任务系统。
- 前台 `bash_run` 使用 `communicate()` 把 stdout、stderr 全部读入内存，最后把拼接结果硬截断到 50,000 字符。
- 截断只保留开头，不告诉 Agent 丢失了多少内容，也没有恢复完整结果的路径；真正的失败原因如果位于输出尾部会永久丢失。

## Reference Findings

本地 Claude Code 实现采用四个值得保留的机制：结构化退出状态、运行中进度、大输出落盘、首尾有界摘要。`learn-claude-code` 的教学实现同样有 50,000 字符截断，但它只是演示基线，不能作为 NaumiAgent 的可靠性交付标准。

本轮只解决“大输出不丢失”。实时流式进度、持久 Shell 会话、命令级只读分类分别属于后续独立能力，避免在一个提交中耦合多个行为变化。

## Options Considered

### 方案 A：提高内存截断上限

改动最小，但仍会丢失输出，并随上限提高放大内存和上下文压力。拒绝。

### 方案 B：所有输出直接写入模型上下文

证据完整，但大日志会迅速耗尽上下文并拖慢 UI。拒绝。

### 方案 C：受管日志 + 首尾摘要

子进程启动前创建权限受限的日志文件，stdout 和 stderr 按实际发生顺序合并写入。完成后，小输出原样返回并删除临时日志；大输出保留日志，只读取固定大小的首尾片段并返回字节数、退出码和完整路径。采用此方案。

## Architecture

### ShellOutputStore

新增一个只负责前台 Shell 输出的轻量存储组件：

- 输出目录由 Engine 注入，生产环境位于 runtime data directory 下，不污染工作区。
- 使用不可预测且独占创建的 `shell-<timestamp>-<random>.log` 文件名，权限为仅当前用户可读写。
- 只管理本目录内、符合命名规则的普通文件；清理时拒绝符号链接和目录外路径。
- 小于等于 50,000 字节的输出在生成结果后删除。
- 大输出保留 7 天且最多保留最近 100 份；活跃命令的日志不会参与清理。
- 无论正常退出、非零退出还是超时，已产生的输出都进入同一结果格式。

### BashRunTool Execution

1. 校验命令、超时和工作目录；工作目录必须位于 workspace root 内。
2. 分配受管日志，以二进制写模式传给 subprocess；stderr 合并到 stdout，以保留发生顺序。
3. 等待命令结束；超时或取消时沿用现有进程树回收。
4. 根据日志字节数生成结果：
   - 小输出：完整解码，删除日志。
   - 大输出：读取开头 24,000 字节和结尾 24,000 字节，中间显示明确的省略统计，保留完整日志。
5. 返回中文结果头，始终包含工作目录、退出码或超时状态、输出字节数；大输出额外包含完整日志路径。

这里的“省略”只是上下文摘要，不是证据丢失；完整日志路径是结果契约的一部分。

### Wiring

`create_builtin_tools()` 新增可选 `shell_output_dir`，Engine 显式传入 `runtime_data_dir / "shell-output"`。直接构造 `BashRunTool` 的测试和扩展仍可使用安全的默认临时目录。

## Error Handling

- 空命令、非整数或非正超时返回中文参数错误，不启动进程。
- `cwd` 不存在、不是目录或逃逸 workspace 时返回中文错误。
- 日志创建失败时不执行命令，避免回退到不可恢复的内存截断。
- 日志读取失败时保留文件并返回路径和错误，不伪装为无输出。
- 进程启动失败时删除空日志；删除失败不覆盖原始启动错误。
- 解码使用 UTF-8 replacement，原始字节仍保存在日志中。

## User Experience

小命令保持当前直观体验，例如 `echo hello` 仍直接看到 `hello`。大命令会看到：

```text
Shell 命令执行完成。
- 工作目录：...
- 退出码：0
- 输出：120000 字节
- 完整输出：.../shell-....log

输出摘要（开头 24000 字节）：
...

...（中间省略 72000 字节，完整内容见上方日志）...

输出摘要（结尾 24000 字节）：
...
```

用户和 Agent 可以直接用 `file_read`/`read` 分段检查日志；UI 无需新增专用协议即可闭环。

## Verification

- 单元测试覆盖空输出、小输出、恰好阈值、大输出首尾、非 UTF-8、非零退出、超时、工作区逃逸、日志创建失败和安全清理。
- 真实场景生成超过 50,000 字节且尾部带唯一错误标记，证明摘要包含尾部标记、完整日志字节数正确、日志可重新读取。
- 只运行 Shell/内置工具相关 pytest、对应 Ruff 和 import smoke，不运行全量测试。

## Deferred Work

- Shell 实时进度事件与 UI 动态展示。
- 自动把超过阻塞预算的命令迁移到 `background_run`。
- 持久 Shell 会话及 `write_stdin`。
- 精确的只读命令分类与无确认并发执行。
