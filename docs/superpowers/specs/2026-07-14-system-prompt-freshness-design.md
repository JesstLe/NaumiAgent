# 系统提示词知识时效设计

## 问题

NaumiAgent 已在每轮 Harness 快照中注入可信本地时间，并会动态列出工具与 Skill；但基础系统提示词没有明确区分稳定知识和易变化信息，也没有要求模型在回答“最新/当前”问题前验证来源。另一方面，生成提示词标记固定为 `sections-v1`，若直接升级标记而不兼容旧标记，旧会话中的生成提示词会被误判为用户自定义内容，从而无法自动刷新。

## 目标

1. 对版本、价格、法规、新闻、模型能力、API 行为等易变化信息强制执行当前证据验证。
2. 保持当前日期时间只存在于每轮临时 Harness 快照，不把会变旧的时间戳写入持久化基础提示词。
3. 旧会话中的 Naumi v1 生成提示词可自动升级为 v2。
4. 用户主动设置的自定义 system prompt 永远不被生成器覆盖。
5. 提示词继续使用动态工具发现，不内联容易老化的完整工具或模型清单。

## 方案

### 版本与识别

当前生成标记升级为 `<naumi_system_prompt version="sections-v2">`。`is_generated_system_prompt()` 不再只匹配当前常量，而是严格识别位于 prompt 开头的 Naumi 生成标记版本格式；这样 v1、v2 及未来版本都可刷新，同时普通文本、正文中的标记引用或伪造的不完整前缀不会被当作生成提示词。

### 知识时效规则

新增 `## Knowledge Freshness` 段，要求模型：

- 先判断事实是否可能随时间变化；
- 对当前、最新、最近、今日、现任、版本、价格、计划、法规、模型/API 能力、兼容性和新闻等内容，优先检查当前工作区源码、锁文件、配置、运行时元数据或权威网络来源；
- 不把训练记忆、静态提示词或旧对话摘要当成当前证据；
- 回答中说明验证时间或依据，无法验证时明确标记未验证和可能过时；
- 对用户指定页面、论文或数据集读取原始内容，不凭标题或记忆代答。

该段不写死供应商、模型、工具版本或日期，因此规则本身不依赖频繁人工更新。

### 动态事实边界

- 当前日期、时间和时区继续由 `HarnessContextAssembler` 每轮注入。
- 工具与 Skill 可用性继续由运行时注册表注入或通过工具发现获取。
- 工作区、权限模式继续由 `PromptAssemblyInput` 动态构建。
- 基础提示词只保存稳定的决策规则，不保存易变化事实快照。

## 错误与安全边界

- 用户自定义 system prompt 不含合法 Naumi 生成标记，因此不会被自动替换。
- 只识别 prompt 开头完整、受限版本字符集的 Naumi 标记，避免用宽泛子串判断任意内容。
- 权威来源不可用时允许继续回答稳定知识，但必须明确区分已验证事实与推断。
- 时效规则不要求所有问题都联网；稳定事实优先使用现有上下文，避免无意义搜索。

## 验证

相关单元测试必须证明：

1. 默认提示词使用 v2 标记并包含知识时效规则；
2. v1 生成标记仍被识别；
3. 类似但不完整的标记不被识别；
4. Engine 加载 v1 旧会话后同时刷新活动消息和完整历史；
5. 用户自定义提示词保持不变；
6. 基础提示词不包含构建日期或当前时间，动态时间仍来自 Harness 测试；
7. 提示词长度保持受控。
